# Forbidden File / Directory Policy

このディレクトリは、CI において
「リポジトリに存在してはいけないファイル・ディレクトリ」を
検出するためのルールを定義する目的で使用されます。

## 目的

- ビルド成果物や一時生成物が誤ってコミットされていないかを検知する
- CI 上で不要・有害なディレクトリ構造を早期に発見する
- 人手によるレビューでは見落としやすいルールを自動化する

## forbidden.txt に含めるもの

以下のような **「存在してはいけない」もの**を記載します。

- ビルド成果物
  - `build`
  - `dist`
- サードパーティ依存の生成物
  - `node_modules`
- 明示的に禁止したいディレクトリやファイル

これらは 1 行につき 1 項目として `forbidden.txt` に記載します。

## forbidden.txt に含めないもの（重要）

以下のような **開発環境や CI 実行時に自動生成されるもの**は、
存在していても問題ないため、`forbidden.txt` には含めません。

- Python 実行時に自動生成されるもの
  - `__pycache__`
  - `*.pyc`
- `.gitignore` によって管理外となっている一時ファイル
- CI 実行中に一時的に生成されるキャッシュ

これらを禁止対象に含めると、CI が不安定になり、
開発者の意図しない失敗を引き起こす可能性があります。

## 運用上の注意

- `forbidden.txt` を更新する際は、
  「本当に存在してはいけないものか？」を必ず確認してください。
- `.gitignore` に含まれているかどうかは、判断の重要な基準です。
- CI で検出された場合、CSV（検出結果）が artifact として保存されます。
  内容を確認し、ルール修正が必要かどうかを判断してください。
